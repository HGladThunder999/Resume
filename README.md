## Resume Hosting using a Static Site Generator

The purpose of this README is to explain how to create a resume using a static site generator and host it using GitHub Pages. Static site generators allow you to create a site that requires no back-end or server side programming.

The project in the repository makes use of Hugo static site generator. Hugo is a static site generator written in the Go Language. It is used to generate a site within seconds.

Git, a version control system was used to manage the different versions of the project. Moreover, Git allows us to work with GitHub via the command line. Most of the code uploaded to the repository was done using Git in the command line. 

In order to edit and create `.toml` and `.md` files VSCode was used. The `.toml` file which is the configuration file of the website was edited using VSCode in order to make use of the theme `ananke` which is a Hugo theme.

GitHub is a forge, which are websites used to collaboratively work on projects. A component of Github is GitHub Pages which is used to deploy the code in the repository. The deployed code by default is hosted on `https://GITHUBUSERNAME.github.io/PROJECTNAME/`.

Throughout this document, the principles covered by _Andrew Etter_ in their book _Modern Technical Writing: An Introduction to Software Documentation_ are followed. Furthermore the guidelines listed by _William S. Pfeiffer_ in their book _Technical Communication_ are also incorporated in the sets of instructions.

## Set of Instructions 

This section is divided into subsections that are referred to as topics by _Andrew Etter_ in their book _Modern Technical Writing: An Introduction to Software Documentation_. It makes it helps reduce duplication of content. 

##### Generating the basic structure of the website

1. Head over to the command prompt or the terminal.
2. Create a new directory to store the files related to the project:
```
mkdir <directory>
```
3. Navigate to the directory you wish to store the files generated using
```
cd <dirName>
```
4. Type the following command in order to generate a hugo project:
```
hugo new site quickstart
```
##### Changing the name and URL of the website

1. Open the `.toml` file manually, using VSCode

`code <FILE.toml>`

2. Change the `Title` to the name you prefer.
3. Change the `BaseURL` to `https://GITHUBUSERNAME.github.io/PROJECTNAME/`.

##### Generating content

1. Head over to command prompt on windows or the terminal on mac
2. Change the directory to directory where the hugo project is supposed to be kept
```
cd <directory>
```
3. Navigate to the `quickstart` folder using the command:
```
cd <directory>\quickstart
```
4. Create a post using the following Command. As Andrew Etter states that lightweight markup in a	text editor	is	faster, easier, and safer. Hugo also makes use of markdown for the posts
```
hugo new content content\posts\NEW.md
```
##### Editing the generated posts and previewing the changes

1. Navigate to the `posts` folder
```
cd content\posts
```
2. Open the file you want to edit using VSCode
```
code <NEW.md>
```
3. Head back to the `quickstart` folder and type the following command:
```
hugo server -buildDrafts
```
4. Preview the website using the command:
```
hugo server
```
##### Generating the website files

1. Navigate to the `quickstart` directory
2. run the following command:
```
hugo
```
##### Adding the files to the GitHub repository

1. Navigate to the `quickstart` folder
2. Run the `hugo` command to generate the website files
3. Rename the new `public` folder generated by hugo:
```
ren public docs
```
4. Navigate back to the quickstart directory and type the following
```
git add .
git commit -M "<Message>"
git push
```

##### Hosting the site using GitHub Pages

1. Go to the repository where you have uploaded all the files.
2. Click the `settings` tab under the repository name.
3. Click on the `pages` section in the code and automation sidebar of the settings page
4. Under "Build and deployment", under "Source", select Deploy from a branch.
5. Under "Build and deployment", use the branch dropdown menu and select main.
6. Use the folder dropdown menu to select `\docs`.


The instructions listed in the above topics adhere to the guidelines listed by _William S. Pfeiffer_ in their book _Technical Communication_

## FAQ
1. What is Markdown?

Markdown is a lightweight markup language that you can use to add formatting elements to plaintext text documents. Lightweight markup languages are languages that are used to define the structure of a document and are easy to learn

2. Do I need coding experience?

Coding experience is not required, However a basic knowledge of git is required if you wish to contribute to the project

3. Why use Hugo?

Because it is fast and flexible

4. Can I host this somewhere else?

Yes, This can be done by uploading the public folder generated by running the `hugo` command. Also you may upload that folder on the following:
- Hostpapa
- Netlify
- wix
- siteground
- etc.

5. What if GitHub Pages is not working?
Make sure you are in the correct branch from the pages option of the settings of your repository for this site `main` was used, and make sure the correct folder is selected for this website specifically `\docs` was used
